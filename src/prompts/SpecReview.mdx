### TDD Compliance
- Tests were written BEFORE implementation (reject if tests came after)
- Unit tests exist for new code
- Integration tests exist for new user-facing features
- Test quality: edge cases, error paths, not just happy path

Flag ALL spec compliance issues. Be thorough.

Severity levels:
- "critical" — Spec violation that breaks core architecture or project identity
- "major" — Missing spec requirement or architectural deviation
- "minor" — Non-critical deviation (naming mismatch, missing docs)
- "none" — Perfect spec compliance

Use severity "none" only if there are absolutely no spec compliance issues.
